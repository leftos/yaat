# M10.1.5 - VFR airspace boundary respect (Class B / Class C)

**Status:** shipped
**Size:** Medium
**Maturity:** implemented

## Outcome

Solo-training VFR aircraft now respect controlled-airspace entry gates instead of blindly crossing every boundary:

- Class B entry requires an explicit Bravo clearance via `CLBRV` / `CBRV` / `BRAVO`.
- Class C entry requires the pilot's initial contact plus a controller response using that aircraft's callsign. Any successful live command satisfies that gate, and `STBY` / `STANDBY` / `ROGER` / `RGR` exists for a pure radio acknowledgement with no maneuver instruction.
- If the gate is not satisfied and projected entry is imminent, the aircraft inserts an airspace-boundary hold, slows to holding speed, transmits that it is holding outside the Bravo/Charlie, and resumes its original route once the gate clears.
- The behavior is gated by `SoloTrainingMode`; instructor/RPO mode keeps the previous behavior.

## Implementation

- `tools/refresh-faa-airspace.ps1` reads vNAS training scenario summaries across ARTCCs, extracts primary airports, and downloads official FAA AIS Class Airspace ArcGIS data for those idents/classes.
- `src/Yaat.Sim/Data/Airspace/faa-training-primary-class-bc.geojson.br` is the checked-in Brotli fixture for Class B/C airspace at all vNAS training primary airports that have matching FAA Class Airspace records.
- `AirspaceDatabase` loads every plain or Brotli GeoJSON under `Data/Airspace`, parses lateral rings plus vertical floor/ceiling attributes, and finds projected boundary entries from aircraft position, track, altitude, and speed.
- `PilotProactive.TickAirspaceBoundaryRespect` runs post-physics in both local sim and yaat-server rooms, VFR-only and solo-mode-only.
- `AirspaceBoundaryHoldPhase` owns the temporary hold outside the boundary, preserves the original route/targets, and restores them when the entry gate is satisfied.
- `AircraftState.IsClearedIntoBravo` and `AircraftState.HasControllerAcknowledgedInitialContact` are snapshot fields so save/replay state survives round trip.
- `CLBRV` sets the Class B gate. `STBY` / `ROGER` sets the two-way-comms gate without changing flight controls.

## FAA Grounding

- AIM 3-2-3 and 7110.65 7-9-2: VFR aircraft must obtain an ATC clearance before operating in Class B airspace; the accepted phraseology includes "cleared through/to enter/out of Bravo airspace" and "remain outside Bravo airspace."
- AIM 3-2-4 and 7110.65 7-8-4: Class C requires two-way radio communications before entry. A response using the aircraft callsign, including "(callsign) standby," establishes communications; a generic response without the aircraft identification does not.
- AIM 3-2-5 and 7110.65 3-1-12 cover the same two-way-comms concept for Class D, but Class D data/behavior is deferred from this shipped slice.

## Deferred Follow-Up: Remain Outside

Add an explicit controller instruction that denies or delays entry while keeping the aircraft outside controlled airspace:

- Bravo phraseology: "REMAIN OUTSIDE BRAVO AIRSPACE" per 7110.65 7-9-2.
- Charlie phraseology: "(callsign) REMAIN OUTSIDE CHARLIE AIRSPACE AND STANDBY" per 7110.65 7-8-4.
- Delta needs a local implementation decision after Class D data is added; AIM 3-2-5 and 7110.65 3-1-12 support the same concept.

This command should not satisfy the entry gate. It should create or refresh the boundary hold, produce an appropriate pilot response, and keep the aircraft outside until the controller later issues `CLBRV` for Bravo or `STBY` / `ROGER` / another targeted successful command for Charlie/Delta. Decide the canonical command shape before implementation; likely options are a generic airspace-aware form (`ROAS B|C|D`) or specific aliases (`ROB`, `ROC`, `ROD`) if they do not collide with existing commands.

## Known Limits

- The shipped data fixture covers Class B/C records for vNAS training primary airports. Airports without Class B/C records are reported by the refresh script and have no boundary behavior.
- Polygon holes and complex multipart semantics are simplified to independent rings. The current FAA fixture and tests cover the SFO/OAK behavior needed for this milestone.
- Active aircraft phases are not interrupted by the proactive boundary watcher in this slice; free-flight/transit VFR aircraft are the target.
- Class D remains planned, not shipped.
