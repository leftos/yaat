## Context

After M10.1 (and now M10.1.1+), the simulator emits realistic readbacks but the student still has to type canonical RPO commands (`N123AB DM 5000`). M10.2 lets the student speak (or type) natural-language ATC English, parsed back into canonical via the existing `PhraseologyMapper` + `LocalLlmCommandMapper` fallback. This is the smaller of the two "feels alive" pieces; M10.3 adds audible voice on top.

## What ships

The student-controller's input box accepts natural-language ATC, not just canonical. Reuses the existing speech pipeline (PTT тЖТ Whisper тЖТ `PhraseologyMapper` тЖТ `LocalLlmCommandMapper`) тАФ but rewires it from "instructor's pilot voice" to "controller's voice." Both work; a setting toggles which side the local user is playing.

## Files to touch

- `src/Yaat.Client/Services/CommandInputController.cs` тАФ when student types something that is not a canonical command, route through `PhraseologyMapper` first; if that returns a canonical, replace the text and dispatch. Today's behavior is canonical-only.
- `src/Yaat.Client/Services/UserPreferences.cs` + settings UI тАФ `SoloTrainingMode` boolean (off = current instructor topology; on = student plays controller and PTT goes through controller-side parsing).
- `src/Yaat.Client/ViewModels/MainViewModel.cs` тАФ when `SoloTrainingMode` is on, route PTT through the same path. (Today PTT already routes through `PhraseologyMapper` тАФ most of this is just re-labeling and ensuring the input ends up in the controller's input box, not the pilot-readback channel.)
