# M10.1.1 — Ground spawn check-ins (IFR + VFR)

**Status:** shipped 2026-05-02
**Size:** Small (~1–2 days)
**Maturity:** shipped

## Context

M10.1 shipped pilot readbacks plus a 5-second AtParking spawn check-in for IFR aircraft only (gated by `HasFlightPlan`). M10.1.1 extends the spawn-check-in pattern to three more phases (`HoldingShortPhase`, `LinedUpAndWaitingPhase`, `FinalApproachPhase`) and drops the IFR-only gate from AtParking so VFR aircraft also check in. It also introduces a cross-phase `HasMadeInitialContact` umbrella flag that downstream milestones (M10.1.2 airborne check-ins, M10.5 unable-rejection) reuse.

YAAT scenarios spawn aircraft in four states that should "talk on first contact" if a human pilot were flying them: `AtParking` (already covered in M10.1), `OnRunway` (LUAW), arbitrary-on-ground reaching a hold-short via preset taxi, and `OnFinal`. M10.1.1 closes those gaps so a solo student hears each freshly-spawned aircraft check in instead of meeting them silently.

## What ships

Four spawn-state pilot check-ins, each fired once per phase entry (or after a short delay), gated by `SoloTrainingMode`. Both IFR and VFR variants where they differ in phrasing. Plus a new umbrella flag plus a phase-instance flag on `HoldingShortPhase`.

| Phase | Trigger | Template (single line) |
|---|---|---|
| `AtParkingPhase` (revised from M10.1) | 5 s after spawn, any flight rules | `"[callsign] ground, [spoken] [location], with information Alpha, ready to taxi."` |
| `HoldingShortPhase` (RunwayCrossing reason) | `OnStart` | `"[callsign] tower, [spoken] holding short runway [rwy], ready for departure."` |
| `LinedUpAndWaitingPhase` | `OnTick`, 10 s after entry, no `DepartureClearance` | `"[callsign] tower, [spoken] runway [rwy], ready."` |
| `FinalApproachPhase` (spawn-on-final only) | `OnStart`, gated by `!HasMadeInitialContact` | IFR with active approach: `"[callsign] tower, [spoken], [approach-name]."` / VFR or no approach: `"[callsign] tower, [spoken] [N]-mile final runway [rwy], with information Alpha."` |

Assumes ATIS information Alpha is current at all fields; real ATIS modeling deferred.

## State / data model

Three new state items, each placed by lifetime:

1. **`AircraftState.HasMadeInitialContact: bool`** — top-level on `AircraftState`. Cross-phase, set the first time *any* pilot transmission fires (AtParking, HoldingShort, LinedUp, OnFinal — and future airborne check-ins in M10.1.2). Top-level rather than `Ground` because M10.1.2 will set it from airborne phases too. Snapshot-serialized via `AircraftSnapshotDto`.
2. **`HoldingShortPhase._hasAnnouncedReady: bool`** — phase-instance private field. Each hold-short location creates a new `HoldingShortPhase` instance (constructor takes a `HoldShortPoint`), so a fresh instance auto-resets the flag without needing `OnEnd` cleanup. Snapshot-serialized via `HoldingShortPhaseDto`.
3. **`AircraftState.HasAnnouncedLinedUpReady: bool`** — top-level. Plan calls for "never cleared (a single LUAW is one logical event)" — a phase-instance flag would re-fire on a touch-and-go's second LUAW. Snapshot-serialized.

`AircraftGroundOps.HasAnnouncedReady` (M10.1, AtParking) stays where it is — moving it would expand the diff for no benefit.

## Reality vs the original plan text

- **`HasAnnouncedReady` lives in `AircraftGroundOps`, not `AircraftState`** (`src/Yaat.Sim/AircraftGroundOps.cs:75`, mirrored at `AircraftGroundOpsDto.cs:15`). The original plan referenced `aircraft.HasAnnouncedReady`; M10.1 actually shipped with `aircraft.Ground.HasAnnouncedReady`. Keep where it is.
- **`AircraftFlightPlan.IsVfr`** (`src/Yaat.Sim/AircraftFlightPlan.cs:29`) is the existing IFR/VFR gate — available for any IFR/VFR template branching.
- **Spawn-on-final detection already partly exists.** `FinalApproachPhase.SkipInterceptCheck` (`FinalApproachPhase.cs:165`) is set true for spawn-on-final aircraft. Combined with the new cross-phase `HasMadeInitialContact`, that's the gate.
- **`HoldingShortPhase` already pushes a sim-internal warning on `OnStart`** (`HoldingShortPhase.cs:51`, into `PendingWarnings`). That stays — controller-facing notice. The new pilot check-in goes into `PendingNotifications` (the pilot-transmission channel).

## Riskiest part

`FinalApproachPhase` is also entered via the natural arrival sequence (vectored, intercepted, established). Aircraft on that path *should* stay silent on phase entry — they spoke earlier (at parking, or via M10.1.2's airborne check-in once that ships). The `HasMadeInitialContact` gate handles this: any aircraft that taxied / took off / cruised set the flag upstream; only true spawn-on-final aircraft hit `OnStart` with the flag false. The negative test guards this contract.

## Open questions / TODO

- Approach-name spoken form for `BuildOnFinal`: derived from `ApproachId` ("I28R" → "ILS two eight right", "R28R-Y" → "RNAV two eight right yankee"). The mapping logic lives in `BuildOnFinal` for now; if other call sites need it, factor out.
- Distance for VFR `[N]-mile final`: computed from `GeoMath.DistanceNm(aircraft.Position, threshold)` rounded to nearest mile. If the aircraft spawns at 0.3 nm "1-mile final" rather than "0-mile" — clamp to a minimum of 1.
