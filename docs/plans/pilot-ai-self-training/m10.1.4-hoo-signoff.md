# M10.1.4 — Contact / Frequency-change pilot speech

**Status:** shipped
**Maturity:** implemented — see `src/Yaat.Sim/Commands/ContactCommandHandler.cs` and `tests/Yaat.Sim.Tests/Commands/ContactCommandHandlerTests.cs`.

## Context

This milestone originally proposed firing pilot sign-off speech automatically inside `TrackEngine.HandleAccept` (HOO accept). That design was rejected before implementation: in real ATC the radar handoff (HOO + ACCEPT) is a coordination action between sectors and the pilot is unaware of it; the frequency-change instruction ("contact Approach on 125.35") is a separate radio transmission. FAA 7110.65 §7-6-11 (`chap07_sec06.md:101-103`) lists them as distinct controller phraseologies:

> CONTACT (frequency identification),
>  or
> FREQUENCY CHANGE APPROVED.

In solo training mode there's no second human controller to play the receiving sector, so `HOO 3O` auto-accepts and the workflow stalls — the student needs an explicit way to issue the frequency change to the pilot. M10.1.4 supplies it via two new canonical commands.

## What shipped

**`CT [target]`** — contact next controller. Tells the simulated pilot to switch frequency.

- Optional argument disambiguates which position to contact. Three forms, in priority order:
  - **Position callsign** — `CT OAK_TWR` / `CT NCT_F_APP` (exact match). Use this when multiple positions share a STARS scope (e.g. OAK_TWR + OAK_GND on TCP 3O).
  - **Frequency in MHz** — `CT 121.9` / `CT 128.525` (±5 kHz tolerance covers both 25 kHz and 8.33 kHz spacing).
  - **TCP code** — `CT 3O` / `CT 2B` (first match wins; ambiguous when multiple positions share the TCP — prefer callsign or frequency forms in that case).
- No argument auto-resolves:
  1. `aircraft.Track.HandoffPeer` if set (HOO initiated, awaiting/just accepted).
  2. Else `aircraft.Track.Owner` when `HandoffAccepted == true` (post-transfer in solo mode).
  3. Else rejects: `"no handoff target — issue HOO first or specify position"`.
- Idempotent — re-issuing `CT` is allowed (controller may legitimately re-instruct).
- Pilot readback: `[ICAO] {facility} on {freq spelled}, {spoken-callsign}, so long.`
  - Example: `[N123AB] approach on one two five point three five, november one two three alpha bravo, so long.`
- **Does not** mutate `Track.Owner` / `Track.HandoffPeer` / `Track.HandoffAccepted`. Drop is still controller-driven.

**`FCA`** — frequency change approved. VFR dismissal when there's no next sector to contact.

- No arguments.
- Pilot readback: `[ICAO] frequency change approved, {spoken-callsign}.`
- Often paired with a follow-up `DROP`, but the controller types both — no auto-drop.

## Architecture

Both commands flow through the existing `CommandDispatcher` pipeline (transparent path, no phase interaction). New components:

- `src/Yaat.Sim/Commands/ContactCommandHandler.cs` — handler. Reads `ctx.ArtccConfig`, resolves the target (callsign / frequency / TCP), builds the readback, routes via `PilotResponder.RouteRpoTransmission`.
- `src/Yaat.Sim/Pilot/PilotResponder.cs` — two new builders: `BuildContactReadback(aircraft, facilityShortname, frequencyMhz)` and `BuildFrequencyChangeApproved(aircraft)`.
- `src/Yaat.Sim/Pilot/FacilityShortname.cs` — maps a position callsign suffix to the spoken short form (`_TWR`/`_ATCT` → "Tower", `_APP` → "Approach", `_DEP` → "Departure", `_GND` → "Ground", `_CTR` → "Center", `_RMP` → "Ramp", `_DEL`/`_CD` → "Clearance", `_FSS` → "Radio"). Falls back to verbatim callsign for unknown suffixes.
- `src/Yaat.Sim/Pilot/PhraseologyVerbalizer.cs` — `FrequencyToWords(double mhz)` formats per 7110.65 §2-4-16: separate digits, "point" at decimal, truncate after second decimal, drop trailing zeros (`121.5` → "one two one point five", `128.525` → "one two eight point five two", `369.0` → "three six nine point zero").
- `src/Yaat.Sim/Data/Vnas/ArtccConfigResolver.cs` — added `FindPositionByFrequency(mhz)` (5 kHz tolerance) and `FindPositionByTcpCodeAnyFacility(tcpCode)` (no facility scope required).
- `DispatchContext` — gained `ArtccConfigRoot? ArtccConfig` field so handlers can do position resolution without a separate delegate. Five Yaat.Sim and three yaat-server construction sites updated.

Routing for the readback follows the M10.1 pattern (NOT the generic `BuildReadback` hook at `SimulationEngine.cs:1035-1041`, which is solo-only and would skip RPO+flag mode):

- `SoloTrainingMode == true` → `aircraft.PendingNotifications` (default kind).
- `RpoShowPilotSpeech == true` (in RPO mode) → `aircraft.PendingPilotSpeech` (green `PilotSpeech` kind).
- RPO without flag → `aircraft.PendingWarnings` (orange controller-debug text).

## Phraseology grounding

- **Frequency formatting**: FAA 7110.65 §2-4-16 (`chap02_sec04.md:501-516`). Truncate (don't round) after the second decimal; drop trailing zeros.
- **"Frequency change approved"**: FAA 7110.65 §7-6-11 (`chap07_sec06.md:103`).
- **"Contact (facility) on (frequency)"**: FAA 7110.65 §7-6-11 (`chap07_sec06.md:101`) — controller side. Pilot drops the verb on readback ("Approach on 125.35, callsign, so long") which is standard GA/airline pilot phraseology.
- **Digit "nine" vs "niner"**: project rule — **controllers say "niner" (per 7110.65 §2-4-17), pilots may say either**. CT and FCA generate pilot speech, so the current "nine" output is compliant. The same rule will apply to any future controller-side speech generation (TTS or text rendering): switch to "niner" for digit 9 there. The STT parser (`AtcNumberParser.cs:42-44`) already accepts both forms for input.
- **Ambiguous TCP code**: when a TCP code resolves to two or more positions (e.g. `3O` → both `OAK_TWR` and `OAK_GND` on a consolidated STARS scope), the handler **rejects** with a candidate list rather than silently picking the first match. This forces the controller to disambiguate via the callsign or frequency form.
- **FCA pilot readback shape**: pilots do **not** parrot "frequency change approved" verbatim — that's the controller phrase per 7110.65 §7-6-11. AIM 4-2-3 ¶3 specifies that a pilot leaving services advises and terminates contact; the readback is a sign-off (`"[ICAO] {callsign}, good day."`), not a recital.
