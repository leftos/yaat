# M12.7 — Achievements / badges

**Status:** Stub — not yet planned. Lowest priority of M12. Confirm with user before building.

## Why

Self-trainers without an instructor lack the external feedback loop that pushes improvement. Persistent achievements ("first clean session at OAK," "20 arrivals no go-arounds," "beat par on KSFO Heavy Push") give a reason to come back and a sense of progress across sessions. It's frankly gamey, but self-training without it has no momentum.

## Sketch

- Persistent local storage (under `%LOCALAPPDATA%/yaat/achievements.json` via `YaatPaths`) tracking unlocked achievements per user.
- Achievement categories:
  - **Scenario completion** — first clean session per scenario.
  - **Streaks** — N arrivals without a separation finding, M sessions without a phraseology ding.
  - **Par-beating** — beat par by 5% / 10% / 20% on a scenario.
  - **Skill milestones** — first ATIS-only session, first instructor-free hour, first solo with realistic pilot speech (M11.1+) enabled at high realism.
  - **Variety** — sessions across N different airports, exposure to weather complications, etc.
- Toast notification + Session Report ribbon when an achievement unlocks.
- Achievements view in main menu: grid of unlocked / locked badges, hover for the unlock condition.

## Open questions

- Achievement *list* — who curates it? Hard-coded JSON? Author-extensible? Probably hard-coded core list + scenario-defined extras (a custom scenario can declare its own unique achievement).
- Difficulty calibration — "20 arrivals no go-arounds" must be achievable to feel rewarding but not trivial. Needs playtesting.
- Cross-session aggregates — most achievements span sessions. Where does session-completion data persist? Probably same `achievements.json` with a session log.
- Achievements that depend on M11 / M12 features (e.g., "high realism" requires M11.1) — gracefully degrade or hide if those features aren't built / enabled.
- Risk of gaming-the-system — "park 20 aircraft in trivial-traffic mode" — design achievements to require non-trivial scenarios.

## Out of scope (probably)

- Online leaderboards / sharing.
- Account / profile system. Single local user only.
- Achievements for instructor / RPO mode. Solo-training-only.

## Touches

- `src/Yaat.Sim/Achievements/` (new) — definition + evaluation.
- `src/Yaat.Client/Services/` (new) — persistence + toast notification.
- `src/Yaat.Client/Views/` — achievements gallery.
- Session Report — unlock ribbon.

## Pick-up notes

**Confirm intent with user before building.** This is the most "gamey" item in the M11/M12 plans; some users will love it, some will find it patronizing. Probably worth shipping as off-by-default with a clear opt-in. If we build it, ship 5-10 achievements first and see how they land before expanding the list.
