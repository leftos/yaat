# M12.5 — Timeline marker overlay + Aircraft filter

**Status:** Shipped. Reframed mid-flight (steps 5-9 collapsed to 5-7) after we noticed the existing rewind UI in `MainWindow.axaml` already provides scrub controls (jump-to-start, ±15/30s, play/pause, free-scrub slider, Take Control). M12.5 stayed focused on what was actually missing: visual markers on the rail, an aircraft filter, and command markers.

## Why the reframe

The original M12.5 stub proposed a `TimelineScrubberWindow` that would download the recording archive, render snapshot-driven aircraft on a radar, and add scrub controls. While planning step 5, we noticed `MainViewModel.Timeline.cs` already provides `RewindToSeconds`, the buttons exist in `MainWindow.axaml:173-195`, and the radar re-renders against rewound state — so most of the planned work was duplicative.

Real M10.7 gaps M12.5 addresses:

- **No visual cue on the timeline for findings.** You can scrub, but you can't see *where* something interesting happened.
- **No way to filter the timeline to a single aircraft.** The Aircraft tab from M12.4 has a "Show on Timeline" button that needs somewhere to jump to.
- **No command markers.** Knowing when you issued commands during the session is review-grade information you currently can't see at a glance.

## What ships

- **Marker overlay** above the existing `TimelineSlider` showing finding markers from `SessionReport.Timeline` — color-coded by severity (Safety red, Warning amber, Coach blue). Click jumps to that time via the existing `RewindToSeconds`. Hover shows finding title + timestamp.
- **Aircraft-filter** on the marker overlay. The M12.4 Aircraft tab's "Show on Timeline" button calls a new `MainViewModel.ShowAircraftOnTimelineAsync(callsign, spawnedAt)` which (a) sets `TimelineFilterCallsign` so only that aircraft's findings remain, (b) rewinds to the aircraft's spawn time. A "Clear filter" affordance on the timeline bar drops the filter.
- **Command markers** — small ticks for controller commands, sourced from a `MainViewModel`-side ring buffer captured at send time. Hover shows the canonical command + recipient.

## Implementation (delivered)

Step 5 — finding markers:
- `src/Yaat.Client/ViewModels/TimelineMarkerVm.cs` — per-marker view-model with severity-aware brush.
- `src/Yaat.Client/Views/TimelineMarkerCanvas.cs` — custom `Panel` that arranges children by an attached `Time` property against a `MaxTime` bound to `TimelineMaximum`. Edge-inset 8 px matches the slider thumb width.
- `MainViewModel.TimelineMarkers` ObservableCollection + 5s `DispatcherTimer` poll in `MainWindow.axaml.cs` that calls `RefreshTimelineMarkersAsync`.
- XAML wraps the existing `TimelineSlider` in a `Grid` with a 6 px marker rail above it.
- `OnTimelineMarkerPressed` handler walks visual ancestors to find the clicked marker's `DataContext` and rewinds.

Step 6 — Aircraft filter (shipped):
- `MainViewModel.TimelineFilterCallsign` + `ClearTimelineFilterCommand`.
- Aircraft tab button enables when a row is selected; click invokes `vm.ShowAircraftOnTimelineAsync(...)`.
- Marker rail shows a filter pill with a "×" to clear.
- `SessionReportWindow` constructor gains a `Func<string, double, Task>? showOnTimeline` callback so the session-report window stays decoupled from `MainViewModel`.

Step 7 — Command markers (shipped):
- `TimelineMarkerVm` gains a `Kind` discriminator (Finding / Command), `CommandText`, and per-kind `MarkerWidth`/`MarkerHeight` so commands render thinner and shorter than findings.
- `RecordCommandMarker(callsign, canonical)` called from the two `SendCommandAsync` aircraft-target dispatch sites on successful return.
- Bounded ring buffer (`_commandMarkerHistory`, capacity 2000) preserves command markers across polling refreshes; the active filter narrows both finding and command markers.
- Live-add on dispatch so commands appear on the rail immediately, without waiting for the next 5 s poll.

## Reuse

- `MainViewModel.RewindToSeconds(double)` — existing rewind entry point.
- `ServerConnection.GetSessionReportAsync()` — existing poll source.
- `MainWindow` rewind buttons + `TimelineSlider` — untouched; markers overlay above the slider.

## Out of scope

- Separate scrubber window (cancelled — existing in-window UI does the job).
- Snapshot-driven radar rendering independent of the rewind path (rewind already drives the live radar to the past state).
- Phase-transition markers — coverage overlap with finding markers; defer.
