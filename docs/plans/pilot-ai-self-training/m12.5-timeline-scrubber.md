# M12.5 — Timeline scrubber on debrief

**Status:** Stub — not yet planned. Flesh out before implementing.

## Why

The snapshot system (M10-era) already records every second of the session — positions, phases, commands, events. The Session Report (M10.7) consumes it for scoring but doesn't expose it for review. A scrubbable timeline lets the user replay the moment of a finding ("you lost separation at 14:23 — here's exactly what was happening") which is fundamentally more useful for learning than a textual citation.

## Sketch

- New Session Report view: timeline along the bottom; radar / ground display above it.
- Timeline shows:
  - Major events (landings, departures, handoffs) as markers.
  - Findings (severity-colored) as markers — click to jump.
  - User actions (each issued command) as ticks — hover for the canonical command + transcription.
  - Active aircraft count over time as a fill band.
- Scrubbing the timeline replays the corresponding snapshot — radar display shows the world state at that instant. Step buttons for ±1s, ±10s.
- Playback button (1x / 2x / 4x) for continuous replay of a segment.
- Reuses existing snapshot infrastructure (`Simulation/Snapshots/`, `Simulation/Replay/`); no new physics integration.

## Open questions

- UI surface — embed in Session Report panel, or new dedicated debrief window? Probably new window so the radar display can be full-size.
- Does the replay engage live simulation, or is it pure snapshot rendering? Pure rendering — never run physics on replay. (See `feedback_replayto_loop_misuse` — `FastForwardTo` is the right API; never re-loop from start.)
- How do we render commands on the timeline? Per-aircraft swim lanes might get too tall with 10+ aircraft. Maybe: all commands collapsed into one row, click filters per-aircraft.
- Interaction with M12.4 (per-aircraft mini-debrief) — each block links to "show me this aircraft's timeline." Aircraft-filtered view that hides unrelated events.
- Export — does the user want to save / share a timeline screenshot or video? Out of scope for v1.

## Out of scope (probably)

- Editing / annotating the timeline (commentary tracks).
- Diff-ing two sessions side-by-side. Save for M12.6.
- Predictive overlays ("if you had vectored here, this would have happened"). Tantalizing but well beyond v1.

## Touches

- `src/Yaat.Sim/Replay/` — already has `FastForwardTo` / `ReplayOneSecond`. Verify API is debrief-suitable.
- `src/Yaat.Client/Views/` — new debrief window with timeline control.
- `src/Yaat.Client/ViewModels/SessionReport*` — debrief navigation.

## Pick-up notes

This is high-leverage but UI-heavy. Start with a static view (no scrubbing, just markers on a timeline that you click to jump to a snapshot). Add scrubbing / playback in a follow-up once the marker-and-jump flow proves useful. Watch out for the `ReplayFromStartTo` footgun — use `FastForwardTo` and step APIs, never loop from start.
