# M10.0 — TTS pipeline spike (Yaat.SpeechSandbox)

**Status:** shipped 2026-04-25 (validated; branched into `tools/Yaat.SpeechSandbox` TTS tab)
**Size:** Small (~1–2 days)
**Maturity:** shipped

## Context

Before committing to M10.3's full TTS layer, we needed a go/no-go validation of the proposed engine stack (sherpa-onnx + Piper LibriTTS-R + radio DSP) on the actual dev box. The spike's purpose was to catch latency / quality / cross-platform problems early, before any production code depended on the choice.

## What shipped

A runnable TTS test bench inside `tools/Yaat.SpeechSandbox`:

- Loaded the Piper LibriTTS-R medium voice (~75 MB ONNX) via sherpa-onnx's `OfflineTtsConfig` / `OfflineTts.Generate(text, sid, speed)`.
- Synthesized 5 ATC-style utterances at 5 different speaker IDs and saved each WAV.
- Measured per-call synthesis latency.
- Applied a band-pass + soft-clip + squelch-tail radio DSP chain to one synthesized clip and saved a `*-radio.wav` for A/B comparison.
- Played one clip through PortAudio end-to-end.

## Findings

- **Cold-call latency: ~1287 ms.** First synth call is slow — sherpa-onnx eagerly initializes the ONNX session. M10.3 must pre-warm with a throwaway utterance at boot.
- **Warm-call latency: 133–207 ms** — well within the sub-second budget for ~10-word utterances.
- **Speaker variation:** 5 speaker IDs in `[0, 500]` produce clearly distinct voices.
- **Radio DSP** (band-pass 300–3000 Hz + soft-clip + ~150 ms squelch tail) produces a recognizable VHF-aviation effect without sounding broken.
- **Memory footprint:** ~100–200 MB with one model resident — acceptable.
- **Cross-platform:** validated on Windows; Mac / Linux deferred to M10.3 CI matrix.

## Locked decisions

These flow into M10.3's design:
- **Engine:** sherpa-onnx (`org.k2fsa.sherpa.onnx` NuGet, Apache-2.0).
- **Voice model:** Piper LibriTTS-R medium (~75 MB ONNX, 904 speakers via `sid` integer, CC-BY 4.0).
- **DSP:** band-pass 300–3000 Hz, soft-clip distortion, squelch tail ~150 ms (NAudio.Core's `BiQuadFilter` is pure managed and works cross-platform; only audio I/O is Windows-only).
- **Pre-warm at boot:** mandatory; first real readback must hit the warm path.
- **Voice pack distribution:** download-on-demand (mirroring `CudaBackendInstaller` precedent), not bundled.

## Open follow-ups

None — the spike succeeded against all go/no-go criteria. M10.3 implements the production pipeline.
