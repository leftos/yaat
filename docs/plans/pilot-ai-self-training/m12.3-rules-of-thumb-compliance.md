# M12.3 — Rules-of-thumb compliance

**Status:** Stub — not yet planned. Flesh out before implementing.

## Why

M10.7 grades formally-codified separation rules. There's a second tier of "rules of thumb" — operational practices that aren't separation minima but are universally expected: no turns below 400 ft AGL on departure, hold-short readback enforcement, descend-via clearances with restrictions, climb-via on SIDs, etc. Real controllers absorb these as habits; a trainer should grade them.

(Wake spacing on final is already covered by M10.7; don't re-grade it here.)

## Sketch

Initial rule set, all FAA / facility-level guidance:

- **No turns below 400 ft AGL on departure** — 7110.65 §3-9-1 implies, AIM §5-2-7 references; vector before climb-out altitude is a finding.
- **Hold-short readback verification** — if controller issues "hold short of XX" and the pilot readback (post-M11.3 perturbation) doesn't include the runway designator, controller is expected to challenge. Missing challenge = finding.
- **Descent-via and climb-via wording** — if a SID/STAR has restrictions, the controller should issue "descend via the {STAR}" or "climb via the {SID}," not bare altitude. Bare-altitude issuance on a STAR/SID is technically valid (cancels restrictions) but should be deliberate.
- **Frequency check on initial check-in** — if a pilot checks in and the controller doesn't acknowledge altitude (when applicable), finding.
- **Two-way comms before entering Class C / Class B** — M10.1.5 already enforces from the *pilot* side; M12.3 grades the *controller's* delay-to-acknowledge.

Each rule emits a Session Report finding with FAA citation; aggregates into a "rules of thumb" sub-score.

## Open questions

- These rules vary by facility / preference more than 7110.65 minima do — make them configurable per scenario? Per ARTCC? Or hard-code a defensible default set?
- Some rules require parsing the pilot's readback (M11.3 territory). Hold-short verification specifically requires divergence-aware readback grading.
- How does this interact with phraseology grading (M12.2)? Phraseology grades *how you said it*; rules-of-thumb grades *whether the action / verbal element happened at all*. Keep them separate.
- Aggregation — single rules-of-thumb sub-score, or per-rule breakdown? Probably per-rule visible, aggregated for at-a-glance.

## Out of scope (probably)

- Facility-specific local conventions ("we always vector arrivals via FOO before clearing visual approach").
- Wake-related rules — already in M10.7.
- LUAW timing / line-up-and-wait procedure rules — possibly covered by M10.7's runway separation already; check.
