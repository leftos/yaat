# M10.4 — Pilot expectations + proactive-after-silence reminders

**Status:** planned
**Size:** Small–Medium (~3–5 days; was originally Medium ~1–1.5 weeks before airborne-spawn moved to M10.1.2)
**Maturity:** drafted — needs revalidation when active. **Trimmed**: the airborne-arbitrary-spawn check-in grid moved to M10.1.2 as part of the VFR-emphasis pull-forward.

## Context

A solo student who forgets to clear an arriving aircraft for the approach gets nothing today — the aircraft just keeps flying its last clearance silently. Real ATC includes pilot prompts: "United 123, with you, ten miles to land 28R" pings the controller. M10.4 introduces a tiny `PilotExpectation` value object on `AircraftState` (NOT a full intent system) and a `PilotProactive` ticker that handles three real-world pilot behaviors: contextual spawn check-in, request-after-silence, pre-handoff sign-off elaboration.

## What ships

- A small `PilotExpectation` record on `AircraftState` computed each tick from current phase + flight-plan state + clearance state.
- A `PilotProactive` ticker that fires reminders in three scenarios (see "Proactive triggers" below).
- Richer phraseology in handoff acknowledgments (next frequency from `PilotExpectation`).

The airborne-arbitrary-spawn check-in grid that originally lived here ships as part of M10.1.2.

## Files to touch

- `src/Yaat.Sim/Pilot/PilotExpectation.cs` *(new)* — small record:
  ```csharp
  public sealed record PilotExpectation
  {
      public ExpectationKind Kind { get; init; }   // OnRamp, ReadyForTaxi, HoldingShort,
                                                   // EnRouteFiledRoute, ApproachExpected,
                                                   // EstablishedOnApproach, LandedTaxiingIn
      public long? UnsatisfiedRequestSinceMs { get; init; }
      public string? PendingRequestKind { get; init; }  // "taxi", "takeoff", "landing"
  }
  ```
  Snapshot-serialized. Computed each tick by `ExpectationUpdater` from current phase + `HasFlightPlan` + `ActiveSidId` / `ActiveStarId` / `DestinationRunway` + clearance state (`DepartureClearance`, `LandingClearance`, `ActiveApproach`).

- `src/Yaat.Sim/Pilot/PilotProactive.cs` *(new)* — ticked from `SimulationEngine` post-physics:
  - **Holding short waiting for takeoff > 60 s with no `DepartureClearance`**: push a "ready for departure 28R, callsign" reminder.
  - **Inside 10 nm of destination on a STAR with no `ActiveApproach`**: push a "with you, ten miles to land 28R" check-in (only fires once per arrival).
  - **At parking with flight plan, no `DepartureClearance`, > 120 s since last reminder**: push "ready to taxi" again.

- `src/Yaat.Sim/Pilot/PilotResponder.cs` — extend templates to use `PilotExpectation` for richer phraseology (e.g., handoff acknowledgment includes the next frequency from `ExpectationUpdater`).

- `src/Yaat.Sim/Simulation/SimulationEngine.cs` — call `ExpectationUpdater.Update(ac)` and `PilotProactive.Tick(ac)` after physics, before broadcast.

## Riskiest part

Spamming the terminal. Each proactive request must have a per-request timestamp gate so it fires once per situation, not once per tick. Aggressive cap: a single aircraft cannot generate more than 1 proactive line per 30 s.

## Verification
