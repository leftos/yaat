# M11.7 — Pilot reports (PIREPs)

**Status:** Stub — not yet planned. Flesh out before implementing.

## Why

Pilots routinely volunteer ride reports, cloud tops, turbulence, icing. The controller is responsible for relaying these to traffic ("AAL222, ride reports indicate light chop FL240, smooth at FL280"). Today the simulator has no weather-driven pilot speech, so this entire workflow doesn't exist. PIREPs are also a soft cue that the radio is alive — pilots are real people exchanging real information, not just compliance bots.

## Sketch

- Triggered by scenario weather state — `SimulationWorld` already has weather (METAR-driven, wind, ceiling). PIREPs draw from:
  - Turbulence layer altitudes → "light chop FL240, smooth above."
  - Cloud tops → "tops at one zero thousand."
  - Icing layer → "light rime in the climb."
  - Ride reports tied to altitude band the aircraft is operating in.
- Low rate (1-2 per long leg per aircraft); skipped entirely if scenario has no weather complexity.
- New pilot-utterance type built by `PilotResponder`. Goes into the SAY queue like any other speech.
- New controller verb (canonical): `RELAY [callsign] PIREP` — relay the report to a specific aircraft. Or `RELAY ALL PIREP` — broadcast to all on frequency. (Verify against 7110.65 phraseology before naming.)

## Open questions

- Does the scenario format currently support detailed enough weather (turbulence layers, tops by sector) to drive PIREPs? Or do we synthesize from existing fields?
- If a player ignores PIREPs entirely, is that a scoring ding (M12)? Probably yes — relaying significant reports is a 7110.65 duty.
- Should PIREPs accumulate / persist across the session, or fade after some time?
- Are PIREP-related vectors a thing? (Pilot deviates for chop → coordinates a heading.) Probably folds into M11.2 (`RequestRide`) instead of being scored separately.

## Out of scope (probably)

- Real-world PIREP database integration.
- Full UA / UUA PIREP format with all the fields — narrative form only for solo training.
- Modelling actual aircraft response to turbulence (bumping the altitude track).

## Touches

- `src/Yaat.Sim/Pilot/PilotProactive.cs` (or equivalent ticker) — PIREP trigger.
- `src/Yaat.Sim/Weather/` — PIREP-relevant accessors (verify what exists).

## Pick-up notes

Lowest-priority of the M11 items unless the scenario weather model is already rich. If weather is sparse, PIREPs become "scripted flavor text," which has diminishing returns. Check the weather state first; defer if there's nothing real to report.
