# M12.4 — Per-aircraft mini-debrief

**Status:** Stub — not yet planned. Flesh out before implementing.

## Why

The Session Report (M10.7) shows a session-level scorecard. Self-trainers want the granularity below that: for each aircraft that went through the airspace, *what happened*. One line per aircraft, post-departure or post-landing, summarizing the trajectory ("AAL222 landed 28R; vectored 2 nm wide to slot behind UAL45; final speed control good; no findings") or flagging issues ("N123AB departed 28R; vectored below 400 AGL on initial climb"). This is the single highest-leverage feedback addition because it converts session-level abstractions into per-event learning moments.

## Sketch

- For every aircraft that completes its session (lands, departs to handoff, drops off), generate a one-paragraph debrief block.
- Block contents:
  - Callsign + type + operation (e.g., "N123AB — Cessna 172 — departed 28R, handed off NCT").
  - Key metrics: efficiency (M12.1), phraseology score on commands issued to this aircraft (M12.2), any rule-of-thumb findings (M12.3), separation findings (M10.7) where this aircraft was involved.
  - Coaching note: one sentence summarizing what stood out — positive or negative ("Smooth descend-via execution" or "Late base turn — extended downwind by 1.5 nm").
- Displayed inline in the Session Report under each aircraft row; expandable.
- Generated live (block appears as soon as the aircraft completes) or batch (all blocks generated on session end)? Probably live — adds dopamine and lets the trainer adjust in-session.

## Open questions

- "Aircraft completes" trigger — landing is clean; departure handoff is clean; what about an aircraft handed off mid-cruise to a sector outside our airspace? Same handoff trigger.
- Coaching note generation — template-driven from the findings, or freeform from a small LLM? Template-driven is safer and deterministic.
- How long does the debrief block stay visible? Permanently in the Session Report, or fade after some time? Probably persistent.
- Interaction with M12.5 (timeline scrubber) — block is the summary, scrubber is the detail. Cross-link from each block to the relevant time range on the timeline.

## Out of scope (probably)

- Multi-aircraft narrative ("here's how you sequenced these three"). Per-aircraft only.
- Generated voice debrief / TTS narration. Text-only.
- Comparing to other sessions / historical averages — that's M12.6 (par).

## Touches

- `src/Yaat.Sim/Session/` — new `AircraftDebrief` type aggregating data from existing findings.
- Session Report UI — new per-aircraft expandable row.
- Possibly `src/Yaat.Sim/Aircraft/AircraftState.cs` — completion hook.

## Pick-up notes

This is probably the single highest-value item in M12 and the cleanest to build because it reuses existing M10.7 findings as data sources. Coaching note is the only piece that needs new logic — start with pure template substitution (highest-severity finding → one-line summary), then iterate on what reads naturally. Don't try to be clever with NLG.
