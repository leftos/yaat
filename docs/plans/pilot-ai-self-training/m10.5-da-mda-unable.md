# M10.5 — DA/MDA contingency + "unable" rejection

**Status:** complete
**Size:** Medium (~1 week)
**Maturity:** implemented

## Context

Two genuine pilot autonomy behaviors that improve training fidelity. The student-controller forgetting to clear a landing should produce a real consequence (the aircraft goes around at minimums); a controller issuing a command the aircraft can't execute should produce a vocalized rejection rather than silent failure.

## What ships

Two genuine pilot autonomy behaviors using a **warn-early-then-miss-at-DA** pattern:

1. **Missed approach with early warning.** When `FinalApproachPhase` descends through `DA + 1000 ft` (or `MDA + 1000 ft` for non-precision) and the aircraft has no landing clearance, the pilot radios `"[N123AB] ... approaching minimums, no landing clearance."` in solo training for Tower/Approach positions, with the existing controller-facing warning path preserved for RPO mode. If the controller issues a landing clearance after the warning and before minimums, the approach continues. When the aircraft reaches the published DA/MDA without a clearance, `FinalApproachPhase` triggers the existing go-around helper with reason `"no landing clearance at minimums"`, preserving the normal `GoAroundPhase` / missed-approach sequencing. Approaches without a published `MapAltitudeFt` keep the prior 1 NM warning and 200 ft AGL no-clearance fallback.

2. **"Unable" responses for impossible commands.** Live solo-training `SendCommand` failures now carry the rejected canonical command type out of `CommandDispatcher`. `SimulationEngine` checks the command registry's `ProducesPilotUnable` bit and queues a delayed pilot readback such as `"[N123AB] ... unable, Command requires VFR aircraft..."` only for pilot-addressable commands. Parse failures, unsupported commands, diagnostic/admin commands, replay/deferred dispatch, and instructor/RPO mode remain silent beyond the normal command failure result.

## Riskiest part

Missed-approach autonomy can hide a student's mistake (they forgot to clear to land — sim quietly goes around). The warning is therefore a radio-visible pilot transmission before minimums, and the go-around uses the existing go-around pilot transmission at minimums. Both are delayed through the normal solo pilot transmission queue so they are heard as frequency events rather than hidden notifications.
