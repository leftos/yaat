# M11.4 — Stepped-on transmissions

**Status:** Stub — not yet planned. Flesh out before implementing.

## Why

When two aircraft key up simultaneously the controller hears noise and has to recover: "station calling, say again" / "last two transmissions, go ahead one at a time." Today the frequency queue (M10.3.5) serializes everything cleanly — no collisions, ever — so this workflow never gets exercised. It's a small but distinctive ATC skill.

## Sketch

- Rare event (~1% of transmissions, configurable per scenario): when a pilot is about to start a SAY transmission, check if another pilot is also about to start within a small window (~500 ms). If yes:
  - Both transmissions garble — neither plays as audio; SAY transcript shows `[unintelligible — two stations]`.
  - Both aircraft enter a "waiting on controller" state: they retransmit only when the controller says "station calling, say again" or addresses one by callsign.
- New canonical command: `SAYAGAIN [callsign]` — addresses an unintelligible transmission. Without callsign, replays the most recent garble in turn order.
- `FrequencyState` handles the resolution — both pilots wait in the queue until released.
- Probability scales inversely with frequency activity (more aircraft = more chance of stomping). May naturally fall out of the existing rate-limit model.

## Open questions

- Does the existing `FrequencyState` queue model accommodate "two transmissions arriving in the same window"? Probably not as-is; needs a small lookahead.
- Do we render any audio for the collision (white noise / squeal) or just the terminal `[unintelligible]`? Audio is more realistic but more work.
- Should controller transmissions get stepped on too? (If controller types while a pilot is talking — currently no concept of "frequency in use." May be a tangent.)
- Scoring (M12): is "stepping on yourself by talking over a pilot" a phraseology ding? Probably yes but small.

## Out of scope (probably)

- Garbled-by-static (range-related dropouts, terrain shadow). Out of scope without modelling radio range.
- Three-way collisions — pairwise is the realistic mode.

## Touches

- `src/Yaat.Sim/Pilot/FrequencyState.cs` — collision detection + recovery state.
- `src/Yaat.Sim/Pilot/PilotResponder.cs` — withhold transmission until released.
- `src/Yaat.Sim/Commands/` — new `SAYAGAIN` command + handler.
- `src/Yaat.Client/` — terminal rendering for `[unintelligible]` SAY entries.

## Pick-up notes

Low-priority flavor item — verify with the user before investing. The training value is real but narrow, and the implementation touches the frequency queue which is already a load-bearing system. If we do build it, ship the text-only version (no audio garble) first.
