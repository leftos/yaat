# CRC reference docs (cached)

Local mirror of the vNAS **CRC Controller Manual** so we can read it offline and pin
YAAT's emulation against a known-good snapshot. These are reference material for designing
YAAT's CRC display emulation — not YAAT's own user docs (those live in `USER_GUIDE.md`).

## Source

- **Upstream**: <https://docs.virtualnas.net/crc/overview/>
- **Fetched**: 2026-06-12
- **Platform**: Material for MkDocs (rendered HTML only — no raw-markdown endpoint, source
  repo is private). The previous upstream was a Docsify SPA serving `*.md` directly; the
  migration is why this mirror is now generated by a converter rather than a verbatim copy.

## Refresh tool

```bash
uv run tools/refresh-crc-docs.py            # refresh CRC manual + Data Admin
uv run tools/refresh-crc-docs.py --section crc
uv run tools/refresh-crc-docs.py --list     # print the page plan and exit
```

[`tools/refresh-crc-docs.py`](../../tools/refresh-crc-docs.py) fetches each page, extracts
the `<article>` body, converts it back to Markdown, downloads every referenced image, and
rewrites links. Specifically it:

- flattens Material admonitions (`!!! info` / `!!! warning`) to emoji-prefixed blockquotes,
  since GitHub does not render MkDocs admonition syntax;
- converts `<figure>` + `<figcaption>` to a Markdown image followed by an italic caption;
- downloads `<section>/img/**` into the local `img/` tree and rewrites `src` to relative;
- rewrites cross-page links to the local `.md` files (including `../vnas-data-admin/…` and
  the separately-mirrored `../vtdls/vtdls.md`).

When upstream changes, re-run the tool and bump **Fetched** above.

## What's NOT mirrored

- The MkDocs site shell (HTML, theme CSS/JS, search index) — not content.
- **vTDLS** — already mirrored separately at [`../vtdls/`](../vtdls/) (from its own Docsify
  site at `tdls.virtualnas.net/docs`); cross-links from these pages point there.
- CRC-client-local behavior that has no server-side analogue (keyboard mappings, DCB
  brightness, secondary windows, browser tabs). It's documented in these pages for context
  but is rendered entirely inside the student's CRC, not driven by YAAT.
